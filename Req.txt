Requirement 2: Validation Rules

The API must validate input data.

firstName

 Invalid

json
{
  "firstName": ""
}


Response:

text
400 Bad Request


Email Format

Invalid

json
{
  "email": "abc.com"
}

Response:

text
400 Bad Request


Duplicate Email

If same email already exists:

```json
{
  "email": "john@test.com"
}
```

Response:

```text
409 Conflict
```

---

## Password Match

❌ Invalid

```json
{
  "password": "abc123",
  "confirmPassword": "xyz123"
}
```

Response:

```text
400 Bad Request
```

---

## Phone Number

Must be exactly 10 digits.

✅ Valid

```text
9876543210
```

❌ Invalid

```text
98765
abc1234567
```

---

## Age

Allowed:

```text
18 - 150
```

❌ Invalid

```text
17
151
```

---

## Address

Must be present.

❌ Invalid

```json
{
  "address": null
}
```

---

## Invalid ID

Example:

```http
GET /api/users/abc
```

Response:

```text
400 Bad Request
```

---

## User Not Found

Example:

```http
GET /api/users/999
```

Response:

```text
404 Not Found
```
